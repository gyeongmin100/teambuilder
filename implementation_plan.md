# Implementation Plan - TeamBuilder

## 1. 개요
`아이디어.md`를 바탕으로 사용자가 입력한 비정형 데이터를 GPT로 분석하고, 정해진 규칙에 따라 팀을 자동 배정하는 SaaS를 구현합니다. `3주차_aistylist`의 아키텍처(Cloudflare Pages + React + OpenAI + Polar)를 계승합니다.

## 2. 기술 스택
- **Frontend:** React 19 (Vite), Tailwind CSS, Lucide React
- **Backend:** Cloudflare Pages Functions
- **AI:** OpenAI GPT-4o-mini (텍스트 정규화 및 스키마 추출)
- **Payment:** Polar.sh (SaaS 수익화)

## 3. 핵심 기능 상세
### 3.1 텍스트 정규화 (GPT)
- 사용자의 자유 서술형 소개글을 분석하여 다음 항목 추출:
  - 선호 역할 (리더, 분석, 실행, 발표 등)
  - 협업 스타일 (직설적, 우회적, 마감 중시 등)
  - 숙련도/기술 스택

### 3.2 팀 배정 알고리즘 (Deterministic)
- 동일한 입력에 대해 항상 동일한 결과를 보장하도록 서버 측 JavaScript로 구현.
- **배정 원칙:**
  1. 팀당 인원수 제한 준수.
  2. 역할 균형 최적화 (팀마다 리더/발표자 최소 1명씩 배치).
  3. 성향 다양성/유사성 설정에 따른 가중치 부여.
  4. 제약 사항 처리 (특정 인원 분리/결합).

### 3.3 UI/UX
- 단계별 입력 폼 (설문지 형태).
- 실시간 처리 상태 인디케이터.
- 시각화된 팀 구성 결과 및 PDF/Excel 내보내기 기능(추후 확장 고려).

## 4. 단계별 작업 계획
1. **환경 구축:** `frontend` 폴더 내의 파일을 루트로 이동하거나 구조를 통일하고, 필요한 패키지를 설치합니다.
2. **백엔드 로직 구현:** `functions/api/assign.js`를 작성하여 GPT 프롬프트 설계 및 배정 알고리즘을 구현합니다.
3. **프론트엔드 UI 개발:** 설문 입력 폼과 결과 대시보드를 만듭니다.
4. **결제 연동:** Polar SDK/API를 사용하여 유료 기능을 제한하거나 결제 유도 로직을 추가합니다.
5. **검증 및 폴리싱:** 다양한 시나리오로 팀 배정을 테스트하고 UI를 다듬습니다.
