# 팀(조) 자동 배정 SaaS 아이디어 정리 (구글폼/엑셀 연계 + GPT 분석)

## 0. 한 줄 요약
구글폼으로 응답만 받으면, 자유서술 소개를 GPT로 정규화/추출하고, 설정(유사/다양/역할균형/제약)에 맞춰 팀을 자동 구성해 결과를 시트로 돌려주는 서비스.

---

## 1. 대상 사용자와 실제 사용 흐름(현장형)
### 1.1 대상
- 학교 선생님: 조별 활동, 수행평가, 프로젝트
- 대학교수/TA: 팀플, 캡스톤, 실습
- 기업 HR: 워크숍/해커톤/교육 팀빌딩

### 1.2 기본 플로우(구글폼/시트 연계)
1) 사용자(교사/교수/HR)가 **템플릿 구글폼** 링크 생성/배포  
2) 대상 인원(학생/직원)이 **소개/협업스타일/선호역할** 등을 입력  
3) 응답이 **구글시트에 자동 적재**  
4) 백엔드가 시트를 읽어 **GPT API로 텍스트를 정규화/스키마 추출**  
5) 서버에서 **팀 조합 알고리즘**으로 자동 배정(결정론적)  
6) 결과를 **구글시트/엑셀로 출력**(팀표 + 근거 요약)

---

## 2. 기존 팀짜기 방식의 한계(문제 정의)
- 랜덤 배정이 대부분 → “공정하지만 목적 최적화 불가”
- 엑셀 수작업 → 시간/노동이 큼, 일관성 없음
- 소개글 같은 비정형 데이터는 활용 어려움
- 불만 발생 시 “왜 이렇게 됐는지” 설명이 어려움

---

## 3. 제품 라인 3종(흩어진 아이디어를 명확히 분리)

# A. 단기간 실현형(추천): 과제 기반 팀 자동 설계
## A1. 핵심 컨셉
“사람끼리만” 매칭이 아니라 **과제/목표를 먼저 분석**하고,
과제가 요구하는 **역할 구성이 충족**되도록 팀을 자동 구성한다.

## A2. 왜 차별적인가
- 흔한 매칭: “비슷한 사람끼리/다른 사람끼리”
- 강한 매칭: “과제 요구 역할 → 사람 역할 매핑 → 팀 설계(목표 최적화)”

## A3. 입력(구글폼 권장 필드)
- 필수: 이름/소속/학번(또는 사번)/가능시간(선택)
- 자유서술(핵심):
  - 협업 스타일(갈등 시 행동, 의사소통 방식)
  - 선호 역할(리더/분석/실행/발표 등)
  - 마감 스타일(초반/후반/균등)
  - 과거 팀플에서 힘들었던 점/잘 됐던 점
- 선택형(최소 보강):
  - 발표 선호도(1~5)
  - 리더 역할 가능 여부(예/아니오)
  - 일정 제약(요일/시간대)

## A4. 처리 원칙(실패 방지 핵심)
- GPT는 “팀을 짜지 않는다”
- GPT는 “텍스트를 **정규화/추출/요약**만 한다”
- 팀 조합은 서버에서 **규칙+최적화**로 “결정론적”으로 만든다  
  (같은 입력이면 항상 같은 결과가 나와야 불만 대응이 가능)

## A5. 사용자 설정(교사/교수/HR)
- 팀 구성 목표:
  - 역할 균형 우선
  - 성향 다양성 우선
  - 성향 유사성 우선
- 제약:
  - 팀 크기(3~6)
  - 특정 인원 분리/같이(블랙리스트/화이트리스트)
  - 시간대 호환(가능시간 겹치기)
  - 팀당 발표자 최소 1명 등

## A6. 출력
- 구글시트:
  - 팀 번호, 팀원 리스트
  - 팀별 역할 분포(리더/발표/분석/실행)
  - “배정 근거 요약”(규칙 기반 + 짧은 설명)

---

# B. 중급 스타트업형: 팀 갈등/붕괴 리스크 점수화
## B1. 핵심 컨셉
팀을 짜고 끝이 아니라, 각 팀 조합에 **갈등 리스크 점수(0~100)**를 산출해 경고한다.

## B2. MVP 현실화 전략(라벨 부족 문제 해결)
- 초기는 “휴리스틱 리스크 점수”로 시작:
  - 직설형 다수 + 회피형 소수
  - 리더 과다/부재
  - 마감 후반형 몰림
- 팀 활동 후 “만족도/갈등 여부” 체크를 받아 라벨을 쌓고 고도화

## B3. 출력
- 팀별 리스크 점수 + 리스크 원인(짧게)
- 대체 조합(스왑) 제안

---

# C. 하드코어형: 팀 퍼포먼스 추적 + 재조합 권고
## C1. 핵심 컨셉
팀은 “한 번 배정”이 아니라, 진행 중 데이터를 보고 **재조합을 권고**한다.

## C2. 필요한 연동(난이도↑)
- Slack/Notion/GitHub/Google Docs 등
- 기여도 쏠림, 침묵 인원, 갈등 언어, 일정 지연 신호 탐지

## C3. 특징
- AI 필수성 매우 높음
- 개인정보/보안/권한 이슈가 커서 기업 중심이 유리
- MVP 범위 설정이 매우 중요

---

## 4. 전체 아키텍처(구글폼/시트 중심)
### 4.1 데이터 흐름
[Google Form] → [Google Sheet] → (Webhook/폴링) → [Backend]
→ [GPT API: 정규화/추출] → [조합 알고리즘] → [결과 Sheet/Excel]

### 4.2 설계 원칙(재현성/설명가능성)
- GPT는 구조화(스키마 추출), 서버는 최적화(팀 조합)
- 결과는 항상 “왜 이렇게 되었는지” 설명 가능해야 함
- 입력/출력 로그를 남겨 분쟁 대응 가능하게 함

---

## 5. 추천 로드맵(현실적)
### Phase 1 (MVP: 2~4주 수준)
- A(과제 기반 팀 설계)
- 폼/시트 연동 + 스키마 추출 + 팀표 출력
- 분리/같이/팀크기/역할 최소 조건

### Phase 2 (고도화)
- B(리스크 점수) 추가
- 팀 만족도 수집(라벨 축적)

### Phase 3 (기업 확장)
- C(협업툴 연동) 검토

---

## 6. 결론(냉정 요약)
- 구글폼/엑셀 연계는 현장 침투력 최상
- “단순 성향 매칭”은 약함 → **과제 기반 역할 충족(A)**로 MVP를 잡아야 산다
- 돈을 크게 벌려면 **리스크 점수(B)**가 핵심 차별화 포인트
- C는 강력하지만 어려움(기업형 장기 과제)
